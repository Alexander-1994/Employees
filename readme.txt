Employees - самое первое приложение на React, которое я сделал.

Для него я, спустя время, в коммите "Update" внес изменения:
- классовые компоненты были переписаны на функциональные (с хуками). 
- логика использования состояния была: в компоненте App был единый state (объект). 
    Состояния и функции по их изменению компонентам передавались по иерархии через пропсы. 
    Я использовал redux для создания единого состояния и react-redux для связи с реактом
- создал фейковую базу данных с персонажами - JSON-файл, который запускается json-server'ом 
    вместе с запуском приложения командой npm start на разных портах (благодаря библиотеки 
    concurrently). При удалении/добавлении персонажей происходит и удаление/добавление в базе 
    данных
- форму по добавлению персонажей теперь контролирует библиотека formik и валидирует библиотека yup
- библиотека reselect, использованная в проекте, позволила избежать лишних рендеров компонента со 
    списком сотрудников при фильтрации за счет создания мемоизированного селектора. Т.е. если 
    пользователь кликает на один и тот же фильтр - значение активного фильтра не изеняется. 
    Соответственно и страница не рендерится.
